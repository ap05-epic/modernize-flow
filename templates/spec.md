## 3. Target Architecture
- **Frontend:** Vite + React + TS. One view → one route + one component tree. CSS Modules over theme CSS
  variables; no component library. Login rebuilt for real auth.
- **Backend (FULL):** `@RestController → @Service → SimpleJdbcCall gateway → stored procedure`. Clean DTO JSON;
  no JSP/Struts concepts leak into the API. Generated by springboot-target-kit; the agent fills business logic.
- **Data modes (per feature):** `record` (replay REAL recorded HAR) · `live` (Vite proxy to legacy) ·
  `api` (the new Spring Boot endpoint, FULL). No hand-authored data in any mode.

## 4. Modernization Principles
- Build from SOURCE, verify with the render — never reproduce structure from a screenshot.
- Preserve business semantics; match legacy behavior BEFORE any improvement; document every intentional deviation.
- Real data only; the recorded HAR is the oracle for both pixel parity (frontend) and contract parity (backend).
- Never accept an error/wrong page as a view (capture quarantines them); always navigate from the start.
- One control/slice per iteration; prove it before moving on. Never flatten stateful behavior into naive params
  without documenting it (§6).
